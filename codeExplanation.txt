1. HIGH-LEVEL OVERVIEW
------------------------------------------------------------------------
Our application has a modular structure consisting of these primary files:
- App.jsx: The root component. It manages global dark mode state, configures React Router client-side routes, and decides when to show the Navbar or Get Started views.
- navbar.jsx: The header component containing the brand logo, a dark mode toggle button, and a router "Get Started" Link.
- Landing.jsx: The landing hero component. Includes CTA links routing to the sign-up page, high-end hover micro-animations, and animated glow assets.
- GetStarted.jsx: The sign-up card view. It features visual text watermarks ("THINK", "CREATE"), interactive forms with visibility-toggle inputs, Apple & Google sign-ins, and a custom mini-header for theme management.

These components are styled using Tailwind CSS v4 (with class-based custom variants in index.css) and keyframe animations in App.css.


2. PARENT & CHILD COMPONENTS: SHARING DATA (PROPS)
------------------------------------------------------------------------
In React, components are structured like a tree. "App.jsx" is the parent, while "Navbar" and "Landing" are its children. 

To share data from a parent component down to a child component, we use "Props" (short for properties). Props are like arguments passed into a JavaScript function.

In our code:
1. In App.jsx, we render the Navbar like this:
   <Navbar darkMode={darkMode} toggleTheme={toggleTheme} />

   Here, we pass two props to Navbar:
   - `darkMode`: A boolean variable (true or false) indicating the current theme.
   - `toggleTheme`: A function that switches the theme when called.

2. In navbar.jsx, we receive these props in the function definition:
   export default function Navbar({ darkMode, toggleTheme }) { ... }

   Now, the Navbar component can read the `darkMode` value (to display either a Sun or Moon icon) and trigger the `toggleTheme` function when the user clicks the toggle button.


3. DEEP DIVE INTO REACT HOOKS
------------------------------------------------------------------------
Hooks are special functions in React that let you "hook into" React features like state and lifecycle methods. In this project, we use the two most common hooks: `useState` and `useEffect`.

--- A. useState ---
State is React's memory. When state changes, React automatically re-renders the component to show the updated data on the screen.

In App.jsx, we declare our theme state like this:
  const [darkMode, setDarkMode] = useState(() => {
    const saved = localStorage.getItem('theme');
    if (saved) return saved === 'dark';
    return window.matchMedia('(prefers-color-scheme: dark)').matches;
  });

Let's break down this declaration:
- `useState`: The hook function. It returns an array with exactly two elements:
  1. `darkMode` (the current state value).
  2. `setDarkMode` (a function we call to update that state value).
- Initializer function: Instead of passing a simple true/false value, we pass a function `() => { ... }`. React runs this function once when the app starts to determine the initial state:
  - First, it checks `localStorage` (a simple database built into the web browser) to see if the user previously chose a theme (`saved`).
  - If a theme is saved, it returns `true` if it's 'dark', and `false` otherwise.
  - If no theme is saved, it falls back to the user's system/OS theme setting using `window.matchMedia('(prefers-color-scheme: dark)').matches`.

To toggle this state, we created a helper function:
  const toggleTheme = () => {
    setDarkMode(prev => !prev);
  };
- `prev => !prev` takes the previous state value and returns its opposite. If `prev` was true, it becomes false, and vice versa.

--- B. useEffect ---
Sometimes, we need to perform "side effects"—actions that affect things outside of React, such as changing the webpage title, making network requests, or modifying the browser's DOM (Document Object Model).

In App.jsx, we use `useEffect` to synchronize our state with the HTML document and local storage:
  useEffect(() => {
    if (darkMode) {
      document.documentElement.classList.add('dark');
      localStorage.setItem('theme', 'dark');
    } else {
      document.documentElement.classList.remove('dark');
      localStorage.setItem('theme', 'light');
    }
  }, [darkMode]);

Let's break it down:
- The first argument is a function containing the code we want to run. If `darkMode` is true, we add the class `"dark"` to the root `<html>` element (`document.documentElement`) and save `'dark'` in `localStorage`. If false, we remove the `"dark"` class and save `'light'`.
- The second argument `[darkMode]` is the **dependency array**. React will only run this effect when the values inside this array change. So, every time `darkMode` switches between true and false, this effect runs automatically!


4. CSS & TAILWIND CSS V4 STYLES
------------------------------------------------------------------------
Tailwind CSS v4 allows us to design beautiful interfaces using utility classes. Let's look at the key layout and styling choices we used:

- `min-h-screen`: Makes sure the wrapper element spans at least the full height of the browser window.
- `relative overflow-hidden`: Positions elements inside relative to this wrapper, and hides anything that goes outside of it (preventing ugly horizontal scrollbars).
- `glass-nav`: Our custom glassmorphism style. Combined with Tailwind's utility class `backdrop-blur-md bg-white/70 dark:bg-[#030303]/70`, it creates a frosted glass effect. As the user scrolls, the background content blurs beautifully behind the navbar.
- `sticky top-0 z-50`: Locks the navbar to the top of the viewport when scrolling, and keeps it on top of other content (`z-50`).
- `flex items-center justify-between`: Uses CSS Flexbox to align items horizontally and space them out, putting the logo on the far left and the controls on the far right.
- `p-2.5 rounded-full`: Gives buttons a circular shape.
- `transition-all duration-300`: Smooths out styling changes (like hovers and clicks) over 300 milliseconds so they don't look abrupt.
- `@custom-variant dark (&:where(.dark, .dark *))`: By default in Tailwind v4, the `dark:` styling variant only matches the user's system preferences. We added this custom line in `index.css` to enable manual toggling. It tells Tailwind to apply dark styling when the `.dark` class is manually added to the root `<html>` element.


5. STAGGERED ENTRANCE ANIMATIONS & GRADIENTS
------------------------------------------------------------------------
To give the landing page a premium "wow" factor, we configured custom keyframe animations in `App.css` using the `@theme` directive in Tailwind CSS v4:

- `@theme` block: This is where we write our custom fonts, animations, and keyframes. Tailwind v4 automatically parses this block and generates CSS classes like `animate-slide-up` and `animate-glow`.
- `slide-up` keyframes: Animates elements from a slightly lower position (`translateY(24px)`) and opacity 0, to their natural position and opacity 1. We use a cubic-bezier timing function (`cubic-bezier(0.16, 1, 0.3, 1)`) for a organic, elastic feel.
- Staggered entry classes:
  - `animate-slide-up`: Runs immediately on load. Applied to the main headings.
  - `animate-slide-up-delay-1`: Waits 0.2 seconds before running. Applied to the description paragraph.
  - `animate-slide-up-delay-2`: Waits 0.4 seconds before running. Applied to the action buttons.
  This delay makes the page content "cascade" into view sequentially, which feels highly premium.
- `glow` keyframes: Slowly moves and scales background color circles behind the text, creating a breathing ambient glow effect.
- Gradients:
  - `bg-gradient-to-r`: Directs a color gradient from left to right.
  - `bg-clip-text text-transparent`: Clips the background gradient to only show within the letters of our text, making the text itself look like a gradient.


6. DETAILED FILE BREAKDOWN
------------------------------------------------------------------------

=== src/App.jsx ===
This file coordinates routing and the global theme. It wraps the app inside the `<Router>` and renders:
- `<Route path="/">`: Renders the main homepage layout, including the full-size glass `<Navbar />`, background ambient radial glows, and the `<Landing />` hero block.
- `<Route path="/get-started">`: Renders only the `<GetStarted />` page, hiding the primary navbar as shown in the design.
- State preservation: Because App.jsx sits at the root, the state values (`darkMode` and its toggler) remain fully intact as the user clicks between screens.

=== src/Components/navbar.jsx ===
Our custom navigation header:
- Logo brand: Wrapped inside a `<Link to="/">` component. It scales up and tilts slightly on hover via `transition-all duration-300`.
- Navigation routes: Replaced normal HTML `<a>` tags with react-router-dom's `<Link to="/get-started">` to load the signup screen instantly without refreshing the browser.
- CTA shadows: The CTA button glows on hover in dark mode via custom text/box shadows.

=== src/Components/Landing.jsx ===
Our hero section:
- Animated drift meshes: Absolute-positioned background blobs drift slowly to add spatial depth.
- Staggered entrance: Heading, paragraph, and CTA buttons cascade into view on page load via sliding keyframes.
- Get Started Link: Linked to `/get-started` using `<Link>` for client-side routing.

=== src/Components/GetStarted.jsx ===
Our account creation signup page:
- Form fields: Captures Full Name, Email, and Password values using a React state dictionary (`formData`).
- Visibility toggle: Checks `showPassword` state. Tapping the eye/eye-off icon changes the input's type between `"password"` and `"text"`.
- Background watermarks: Displays huge `"THINK"` and `"CREATE"` text blocks in the background with light transparency.
- Custom mini-header: Features a centered brand logo (linking back to the home route `/`) and a right-aligned theme toggle.

=== src/App.css ===
This stylesheet contains our custom font definitions and Tailwind v4 configurations:
- `@font-face`: Registers local font files (IBM Plex Sans for headings, Elms Sans for body text) so the browser can load and display them.
- `body { transition: background-color 0.5s ease; }`: Makes the transition from light background (neutral-50) to dark background (#030303) smooth and gradual when the user clicks the theme toggle!
